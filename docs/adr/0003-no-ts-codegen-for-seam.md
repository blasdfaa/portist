# 0003 — Форму шва Rust↔TS держим руками, без кодогена

- Статус: Принято
- Дата: 2026-07-03

## Контекст

Форма данных, пересекающих шов Tauri (`invoke`), определена дважды: serde-структуры
в Rust (`ports.rs`, `details.rs`, `docker.rs`) и TS-интерфейсы в
`src/app/core/models.ts`. Шов рантаймовый (`invoke<PortDetails>(...)`), поэтому
рассинхрон имён/типов компилятор не ловит — только рантайм.

Рассматривались `ts-rs` (генерация TS-типов из Rust) и `tauri-specta` (типы +
типизированные invoke-обёртки), чтобы сделать Rust единственным источником формы.

## Решение

Кодоген **не вводим**. Форму держим синхронной вручную.

Причины (цена тулчейна выше налога дублирования для этого проекта):

- Структур всего четыре — `PortInfo`, `PortDetails`, `ContainerInfo`,
  `ContainerPort`. Поля 1:1, меняются редко (обычно — добавление поля деталей).
- `ts-rs` потерял бы русские док-комментарии моделей (они несут смысл полей:
  «0 — неизвестно» и т.п.).
- `ts-rs` **ослабил** бы типы: Rust `protocol: String` → `protocol: string`, тогда
  как сейчас в TS осознанно `type Protocol = "tcp" | "udp"`. Сохранить union можно
  только переведя Rust на `enum Protocol` — цена каскадит за пределы «просто
  генерации».
- `tauri-specta` — крупный buy-in (перестройка всего слоя invoke) ради шва из
  четырёх стабильных структур.

## Последствия

- Форму синхронизируем руками: при добавлении поля деталей правим `details.rs`
  **и** `models.ts` (см. locality-налог — 5+ файлов на новое поле). Rust-тесты
  (`cargo test`) ловят логические ошибки, но не рассинхрон формы TS↔Rust.
- **Известное ограничение:** TS-union `Protocol` строже Rust-контракта
  (`protocol: String`, а у контейнеров — сырьё из Docker `Type`). На практике
  демон отдаёт tcp/udp; союз поддерживаем как инвариант вручную.
- Этот ADR существует, чтобы будущий architecture review **не пере-предлагал**
  ts-rs/tauri-specta как «недостающую генерацию»: ручная форма — осознанный выбор
  под масштаб проекта. Пересмотреть, если число структур/частота правок заметно
  вырастут.
